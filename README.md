# Kindle電子書籍 原稿作成テンプレート

Claude Codeのスキル（スラッシュコマンド）を使い、市場で勝てるKindle電子書籍の原稿を効率的に作成するテンプレートプロジェクトです。

## 特徴

- **複数書籍の同時管理**: `books/` ディレクトリ配下で複数の原稿を並行管理
- **出版社チーム体制の再現**: 編集長・構成作家・ライター・校正者・校閲者・仕上げ担当の6役割をAgent Teamsで並列実行
- **エージェント間の直接通信**: ライターが編集長に質問、校閲者が編集長に問題をエスカレーション等
- **共有タスクリストによる進捗管理**: タスク依存関係で工程順序を自動制御
- **市場戦略の組み込み**: BDF分析・HARM法則・ニッチ戦略を企画段階で自動実施
- **Gitブランチ運用**: 書籍ごとにfeatureブランチで作業し、レビュー完了後にPR
- **品質管理**: 校正→校閲→編集長の3段階レビューを経なければPR作成不可

## セットアップ

```bash
git clone https://github.com/smilior/kindle-template.git
cd kindle-template
```

Claude Codeで開き、スラッシュコマンドを実行するだけで使えます。

## クイックスタート（一番かんたんな使い方）

**1冊まるごと作る場合、コマンドは1つだけ：**

```
/ebook-produce
```

対話で書籍の基本情報（タイトル・ジャンル・ターゲット等）を入力すると、
企画 → 戦略設計 → アウトライン → 全章執筆 → レビュー → 原稿結合 → PR作成まで全自動で実行されます。

### 企画済みの書籍を制作する場合

```
/ebook-produce {slug}
```

`/ebook-init` で作成済みの書籍に対して、アウトライン作成〜PR作成まで一気通貫で実行します。

## 全コマンド一覧

### メイン制作フロー

| コマンド | 説明 | 入力 | 成果物 |
|---------|------|------|--------|
| `/ebook-produce` | **全工程一括実行（推奨）** | 対話で書籍情報入力 | book.yaml〜manuscript.md＋PR |
| `/ebook-produce {slug}` | 企画済み書籍の全工程一括実行 | スラッグ | outline.md〜manuscript.md＋PR |

### 工程別コマンド（個別に実行したい場合）

```
/ebook-init         新規書籍の企画・初期化（対話 → book.yaml + featureブランチ作成）
     ↓
/ebook-outline      アウトライン作成（構成作家 → 編集長レビュー）→ outline.md
     ↓
/ebook-write        章の執筆（ライター × 並列実行）→ chapters/*.md
     ↓
/ebook-review       校正・校閲（校正者 + 校閲者 × 並列 → 編集長統合）★PR前に必須
     ↓
/ebook-build        原稿結合・仕上げ（巻末要素の自動チェック付き）→ manuscript.md
     ↓
/ebook-pr           プルリクエスト作成（全章レビュー済みチェック付き）
```

### ユーティリティ

| コマンド | 説明 |
|---------|------|
| `/ebook-status {slug}` | 書籍の進捗確認（各章のステータス・文字数・チーム情報） |
| `/ebook-list` | 管理中の全書籍一覧 |

## 使い方の例

### 例1: 新規書籍を企画からPR作成まで全自動

```
/ebook-produce
```

1. タイトル・ジャンル・ターゲット等を対話で入力
2. 戦略設計（BDF分析・ニッチ戦略）を確認
3. 以降は全自動：アウトライン → 全章執筆 → 3段階レビュー → 原稿結合 → PR作成

### 例2: 工程を分けて確認しながら進める

```
/ebook-init                        # 企画・戦略設計（対話あり）
/ebook-outline ai-business-guide   # アウトライン作成・確認
/ebook-write ai-business-guide all # 全章執筆
/ebook-review ai-business-guide all # 全章レビュー
/ebook-build ai-business-guide     # 原稿結合
/ebook-pr ai-business-guide        # PR作成
```

### 例3: 特定の章だけ執筆・レビュー

```
/ebook-write ai-business-guide 3   # 第3章だけ執筆
/ebook-review ai-business-guide 3  # 第3章だけレビュー
```

## チーム体制（Agent Teams）

書籍制作はAgent Teamsの仕組みで、出版社のプロフェッショナルチームを再現しています。

| 役割 | メンバー名 | 動作期間 | 責務 |
|------|-----------|---------|------|
| 編集長 | `editor-in-chief` | outline〜build | 全工程の品質監督・最終レビュー |
| 構成作家 | `structure-writer` | outline | アウトライン設計 |
| ライター | `writer-ch{NN}` | write | 章ごとの本文執筆（並列） |
| 校正者 | `proofreader` | review | 誤字脱字・文法・表記ゆれ修正 |
| 校閲者 | `content-editor` | review | 内容の正確性・論理性チェック |
| 仕上げ担当 | `finalizer` | build | 原稿結合・巻末要素チェック |

- チーム名: `ebook-{slug}`（例: `ebook-ai-business-guide`）
- 編集長はアウトラインから原稿結合まで全工程に存続し、一貫した品質判断を行う
- ライターは編集長にアウトラインの疑問を質問できる
- 校閲者は重大な問題を編集長にエスカレーションできる

## プロジェクト構造

```
kindle-template/
├── CLAUDE.md                 # プロジェクトルール定義
├── README.md                 # このファイル
├── books/
│   └── {book-slug}/
│       ├── book.yaml         # メタデータ + 戦略設計（BDF・キーワード等）
│       ├── outline.md        # アウトライン
│       ├── chapters/         # 各章のMarkdownファイル
│       ├── reviews/          # レビューレポート
│       ├── assets/           # 画像・図表
│       └── manuscript.md     # 結合済み原稿（自動生成）
└── .claude/
    └── skills/               # スラッシュコマンド定義
```

## 書籍設計のベストプラクティス

本テンプレートには以下のベストプラクティスが組み込まれています。

### テーマ選定（HARMの法則）

人間の根源的な悩みに該当するテーマを選定：
- **Money** — お金・副業・投資
- **Health** — 健康・美容・メンタルヘルス
- **Ambition** — 夢・キャリア・自己啓発
- **Relation** — 人間関係・恋愛・家庭

### コンセプト設計（BDF分析）

読者の深層心理を3軸で分析し、差別化コンセプトを構築：
- **Belief** — 読者の固定観念を特定し、否定する
- **Desire** — 読者の本当の欲求を満たす
- **Feeling** — 読者の現状の感情に寄り添う

### 原稿構成（3つの壁の突破）

1. **読まない壁** → 序章でBDFに共感し、ベネフィットを提示
2. **信じない壁** → 本編で体験談・データによる信頼構築
3. **行動しない壁** → 終章で行動喚起とバックエンド導線

### 品質基準

- 全体20,000文字前後 / 1章1,500〜2,500文字
- 各章末にブリッジ（次章予告）
- 巻末にレビュー依頼文・読者限定特典・著者プロフィール
- 3段階レビュー（校正者 → 校閲者 → 編集長）

## Gitブランチ運用

```
main ──────────────────────────────── merge ← PR
  └── feature/{slug} ── write ── review ── build ── PR
```

- 書籍ごとに `feature/{slug}` ブランチで作業
- 各工程完了時に自動コミット
- 全章レビュー完了後のみPR作成可能
