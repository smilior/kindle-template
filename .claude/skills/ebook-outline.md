# /ebook-outline — アウトライン作成

書籍のアウトライン（構成案）を作成する。SubAgentで構成作家と編集長を起動する。
引数: 書籍スラッグ（例: `/ebook-outline ai-business-guide`）

## 手順

1. `books/{slug}/book.yaml` を読み込む
2. 以下の2つのSubAgentを**直列**で起動する

### Phase 1: 構成作家（Structure Writer）SubAgent

Taskツールで以下を実行する：
- `subagent_type: "general-purpose"`
- プロンプト内容：

```
あなたはプロの構成作家です。電子書籍のアウトラインを作成してください。

【書籍情報】
{book.yamlの内容をここに展開}

【指示】
以下の形式でアウトラインを作成し、books/{slug}/outline.md に書き込んでください。

1. 書籍全体のコンセプトを明確にする
2. 各章のタイトル、概要（3〜5文）、キーポイント（3〜5個）を設計する
3. 章間の論理的な流れを確保する
4. ジャンルに応じた構成パターンを適用する：
   - ビジネス書: 課題提示 → 解決策 → 実践方法
   - 自己啓発: 現状分析 → 気づき → 変化のステップ
   - 技術書: 基礎 → 応用 → 実践プロジェクト
   - エッセイ: テーマ別のエピソード群
5. 各章は独立して読めるが、全体として一貫した流れを持つこと
6. 最初の章で読者の興味を引き、最後の章で行動を促す構成にすること

【出力形式】
# {書籍タイトル} — アウトライン

## 書籍概要
{概要}

## 想定読者
{ターゲット}

## 全体構成

### はじめに
- 書籍の目的
- 読者が得られるもの
- 本書の読み方

### 第1章: {章タイトル}
**概要**: {章の概要}
**キーポイント**:
- ポイント1
- ポイント2
- ポイント3

（以降、全章分）

### あとがき
- まとめ
- 読者へのメッセージ

【出力先】
books/{slug}/outline.md に Write ツールで書き込むこと。
また、各章に対応する空のchapterファイル（01-chapter.md 〜 NN-chapter.md）を
books/{slug}/chapters/ に作成すること。
各chapterファイルにはフロントマター（chapter番号、title、status: draft、word_count: 0、reviewed_by: []）を含めること。
```

### Phase 2: 編集長（Editor-in-Chief）SubAgent

Phase 1 完了後に起動する：
- `subagent_type: "general-purpose"`
- プロンプト内容：

```
あなたは出版社の編集長です。構成作家が作成したアウトラインをレビューしてください。

【書籍情報】
{book.yamlの内容}

【指示】
books/{slug}/outline.md を読み込み、以下の観点でレビューしてください：

1. 書籍のコンセプトと想定読者に合った構成か
2. 章間の論理的な流れに飛躍や重複がないか
3. 読者を引き込む導入になっているか
4. 結論が行動を促す構成になっているか
5. 各章のボリュームバランスは適切か

問題があれば outline.md を直接修正してください。
修正した場合は、何をどう変えたかを簡潔に報告してください。
修正がなければ「承認」と報告してください。
```

3. 結果をユーザーに報告し、フィードバックがあれば修正する
