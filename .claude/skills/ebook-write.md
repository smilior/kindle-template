# /ebook-write — 章の執筆

ライターSubAgentを並列起動して章を執筆する。
引数: 書籍スラッグ [章番号|"all"]（例: `/ebook-write ai-business-guide 3` または `/ebook-write ai-business-guide all`）

## 手順

1. `books/{slug}/book.yaml` と `books/{slug}/outline.md` を読み込む
2. 引数に応じて対象章を決定する：
   - 章番号指定 → その章のみ
   - `all` → status: draft の全章
   - 未指定 → 次の未執筆章を1つ
3. 対象の各章について、ライターSubAgentを**並列**で起動する

### ライター（Writer）SubAgent

対象章ごとに1つずつTaskツールを起動する（並列実行）：
- `subagent_type: "general-purpose"`
- 複数章の場合は**1つのメッセージ内で複数のTaskツールを同時に呼び出す**こと
- プロンプト内容：

```
あなたはプロのライターです。電子書籍の第{N}章を執筆してください。

【書籍情報】
{book.yamlの内容}

【アウトラインの該当章】
{outline.mdから対象章のセクションを抜粋}

【執筆ガイドライン】
- 文体: {book.yamlのtone}で統一する
- 一文60文字以内を目安
- 章の冒頭: 読者の興味を引くエピソードや問いかけで始める
- 本論: アウトラインのキーポイントを具体例やエピソードを交えて展開する
- 章の結び: 要点のまとめと次章への橋渡し
- 適度に小見出し（##, ###）を使う
- 読者への問いかけを適宜入れる
- 箇条書きを活用して情報を整理する
- 短い段落を心がける（スマホでの可読性）
- 太字（**強調**）を効果的に使う
- 1章あたり3,000〜5,000文字を目標
- 内容の充実度を優先し、水増しはしない

【出力先】
books/{slug}/chapters/{NN}-chapter.md に Write ツールで書き込むこと。
既存のフロントマターを保持し、status を "writing" に、word_count を実際の文字数に更新すること。
```

4. 全SubAgentの完了後、各章の文字数を集計して報告する
